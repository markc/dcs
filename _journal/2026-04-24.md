## Dropped `base.txt`, added `docs/ai.txt`

- `base.txt` (246 lines) was an early plaintext DCS site-generation spec, superseded by the formal `_spec/` triad. Only referenced in one CLAUDE.md layout-tree line.
- **Fetched** `https://aitxt.ing/ai.txt` (162 lines) — the spec format with YAML frontmatter, cascading discovery, distributed context via links ending in `ai.txt`, and the `<link rel="prefetch">` HTML Discovery convention.
- **Authored** `docs/ai.txt` (202 lines) as the new canonical AI-readable spec. Sections: About / Architecture / Features / Capabilities / **Limitations (explicit non-goals)** / Data (structured facts — 23 keys) / Usage Patterns / FOUC / HTML Skeleton / AI Guidelines / Permissions / See Also / Contact. Pulled HTML skeleton, CSS variable contract, and responsive breakpoints from `base.txt`; pulled architecture, color-system, and component facts from `_doc/`; pulled motion/interaction/pin-geometry from `_spec/`.
- **Crucial for AI agents:** the Limitations section is deliberately blunt — "no framework, no build step, no npm package, no component API, no React" — to prevent hallucinated capabilities. The AI Guidelines section tells agents where to look for deeper context (`_spec/` for structured facts, `_doc/` for prose) and when *not* to edit (`base.css` / `base.js` are framework files).
- **Symlinked** `ai.txt → docs/ai.txt` at root (same pattern as `base.css` / `site.css` / `md.js`).
- **Added** `<link rel="prefetch" href="ai.txt">` to `docs/index.html` `<head>` per the aitxt HTML Discovery section — browsers and AI-aware browser extensions pick it up during idle fetch.
- **Deleted** `base.txt`; updated `CLAUDE.md:34` entry.
